Простой пример

Основная команда для отрисовки «x-y графиков» выглядит так: plot(x,y), где x и y — это координаты. Если задать всего одну пару чисел, то будет отмечена одна точка. Но чаще всего передают векторы (должны быть одинаковой длины), и тогда команда отображает все указанные точки из этих векторов, соединяя их между собой прямыми. Например построим график \(y = \sin(x)\) на диапазоне от \(-\pi\) до \(\pi\):

Пошагово:

  1. x = -pi:pi/10:pi;: Создаёт вектор x, который начинается с \(-π\), заканчивается \(π\), и увеличивается на \(\frac{\pi}{10}\) на каждом шаге. В итоге будет создано length(x) → 21 точек.
  2. plot(x, sin(x));: Строит график функции sin(x) в зависимости от x. График будет представлять собой кривую синусоиды на заданном интервале x.
  3. xlabel("x");: Устанавливает название для оси абсцисс (горизонтальной оси) как “x”.
  4. ylabel("sin (x)");: Устанавливает название для оси ординат (вертикальной оси) как “sin (x)”.
  5. title("Простой 2-D График");: Устанавливает заголовок графика как “Простой 2-D График”.

В итоге, будет создан двумерный график функции \(\sin(x)\) на интервале от \(-\pi\) до \(\pi\) (радианы), с подписями осей и заголовком. На большинстве систем команда plot откроет новое окно с графиком:

Базовые улучшения в построении

При создании графиков в Matlab/Octave у пользователя есть множество способов сделать их более понятными, читаемыми и красивыми. Вот несколько советов и методов, которые помогут улучшить предыдущий график:

  1. Использование функции linspace. Может быть полезно использовать функцию linspace(start, end, n); вместо start:increment:end; для генерации независмого вектора x, что позволяет контролировать “разрешение” оси x, задавая нужное количество точек напрямую. Для примера выше, экспериментальным путем, выясним удовлетворяющее число n = 100.
  2. Толщина линии. Для более ясного отображения линии следует увеличить её толщину передав соответсвующую пару <свойство>, <значение> функции plot: plot(..., 'LineWidth', 2, ...);.
  3. Настройка размеров шрифта подписей осей и заголовка графика. Команды xlabel, ylabel - отвечающие за метки осей а также title - отвечающий за заголовок графика принимают опциональный список пар <свойство>, <значение> для изменения свойств объекта. Таким образом, передавая дополнительный аргумент 'Fontsize' и его значение, например 14 в случаи подписей осей: x/ylabel(..., 'Fontsize', 14, ...) и '16' в случае заголовка: title(..., 'Fontsize', 16, ...), мы можем увеличить размер текста повысив читаемость.
  4. Поля вокруг данных. Границы графика не должны касаться границы холста, этого можно избежать добавив простанство вокруг максимальных и минимальных значений x-y и осей:
    • Добавление полей по оси абсцисс: xlim([min(x) - x_margin, max(x) + x_margin]);. Для примера выше подходящим будет: x_margin = 1
    • Добавление полей по оси ординат: ylim([min(y) - y_margin, max(y) + y_margin]);. Для примера выше подходящим будет: y_margin = .5
  5. Форматирование LaTeX. Имея дело с математическимим сущностями, часто небходимо их аннотировать правильным математическим форматированием. В Matlab/Octave для этого можно использовать стандартное форматирование LaTeX. Для вставки текста в формате LaTeX в заголовок, метки осей или другие части графика, следует использовать свойство 'Interpreter' и установить его значение в 'latex'. Это позволяет корректно отображать сложные математические выражения и уравнения прямо на графиках. В нашем случае: title("Graph $y = \sin(x), x \in [-\pi; \pi]$", 'Interpreter', 'latex', ...);. Стоит иметь в виду, замена Graph на русскоязычное слово График приведет к ошибки в связи не возможности обработки интерпретатора не ascii символов.
  6. Сетка. Сетка поможет в точном чтении значений графика, для её добавления: grid on;
  7. Более информативное представление радиан. Чтобы установить деления по оси x в радианах в более информативном виде: \(\frac{\pi}{2}, \frac{\pi}{4}\) и т.д. необходим сделать следующее:
    • xticks([-pi, -3*pi/4, -pi/2, -pi/4, 0, pi/4, pi/2, 3*pi/4, pi]); - устанавливает позиции делений (или меток) на оси x графика;
    • xticklabels({'$-\pi$', '$-\frac{3\pi}{4}$', '$-\frac{\pi}{2}$', '$-\frac{\pi}{4}$', '$0$', '$\frac{\pi}{4}$', '$\frac{\pi}{2}$', '$\frac{3\pi}{4}$', '$\pi$'}); - устанавливает текстовые метки для делений на оси x графика. В данном случае эта команда заменяет стандартные метки делений на оси x на указанные в списке математические выражения в формате LaTeX.
    • set(gca, 'TickLabelInterpreter', 'latex'); - устанавливает интерпретатор для текстовых меток делений текущих осей графика на значение ‘latex’. Когда интерпретатор установлен на ‘latex’, Matlab будет интерпретировать текстовые метки как код LaTeX. Это позволяет отображать математические символы и выражения в формате LaTeX на графике.
    • Для увеличения размера подписей необходимо изменить соответсвующие свойство текущих осей: ax = gca; ax.XAxis.FontSize = 14;. Более подробно1
  8. Увеличение отметок оси y → ax.YAxis.FontSize = 14;

Изображение точек интереса

Часто может потребоваться нахождение и изображение определенных точек интереса на графике, например точек минимума и максимума или корней.

Нахождение и изображение точек минимума и максимума

1) Создание анонимной функции

Здесь:

  • f: Это имя, данное анонимной функции. После её определения её можно использовать, как и любое другое имя функции;
  • @(...): Обозначает входные аргументы функции. В этом контексте анонимная функция принимает один аргумент, x;
  • sin(x): Это выражение функции. Оно указывает, что функция возвращает. В этом случае возвращается \(\sin x\).

2) Нахождение минимума функции

Здесь:

  • Функция fminbnd в Matlab используется для поиска значения x (x_min_val) унимодальной функции f (функции, у которой есть один локальный и глобальный минимум) на указанном интервале при котором достигается минимальное значение y (y_min_val);
  • f - это функция, минимум которой мы хотим найти. В данном контексте это может быть анонимной функцией (например, f = @(x) sin(x);) или любой другой определенной функцией;
  • -pi и pi - это границы интервала, на котором мы хотим найти минимум. В этом случае мы ищем минимум функции f на интервале \([-\pi, \pi]\);
  • [x_min_val, y_min_val] - это результаты работы функции fminbnd. В частности: x_min_val: значение x в пределах интервала \([-\pi, \pi]\), при котором функция f достигает своего минимума; y_min_val: соответствующее минимальное значение функции f. Используя синтаксис квадратных скобок: [x_min_val, y_min_val] - в некотором смысле производится распаковка значений: функция fminbnd возвращает два значения, и мы одновременно присваиваем каждое из этих возвращаемых значений отдельным переменным (x_min_val и y_min_val). В отличии от других языков программирования при предоставлении только одной переменную для вывода, Matlab, как правило, вернет только первое значение (в других же чаще всего вернется «не распакованная» коллекция).

3) Накладывание графических элементов

Команда используется для того, чтобы на текущем графике или окне отображения можно было наносить последующие графические данные без стирания или замены предыдущих.

Отображение корней

Для нахождения корней можно воспользоваться функцией fzero, требующая начальные приближения. В нашем случае: initial_guesses = [-2.5, 0, 2.5]; - приближения корней \([-pi, 0, \pi]\).

Итоговый код отображения корней на графике:

Функция arrayfun применяет заданную функцию к каждому элементу массива. В данном случае к массиву initial_guesses.

Подробнее разберем, что делает следующий код: arrayfun(@(guess) fzero(f, guess), initial_guesses);:

  • fzero(f, guess): Это функция для поиска корня (или нуля) функции. Она пытается найти значение x, такое что f(x) = 0. Функция fzero требует двух аргументов: функции, корень которой вы хотите найти (в этом контексте f), и начального приближения для корня (guess);
  • @(guess) fzero(f, guess) - создание анонимной функциn. Это способ определения простой функции в строке, без необходимости именования или определения ее в другом месте. Эта конкретная анонимная функция принимает один аргумент, guess, и возвращает результат fzero(f, guess);
  • arrayfun(...): Это функция высшего порядка4, которая применяет данную функцию (в данном случае нашу анонимную функцию из предыдущего пункта) к каждому элементу предоставленного массива (initial_guesses). Таким образом, для каждого значения в initial_guesses она будет искать корень функции f, используя это значение как начальное приближение.

В итоге, строка arrayfun(@(guess) fzero(f, guess), initial_guesses); находит корни функции f, используя каждое из значений в initial_guesses в качестве начальных приближений. Результатом является массив корней, соответствующих каждому начальному приближению.

Рассмотрим одновременное отображение нескольких графиков на примере стандартных преобразований функций.

Растяжение и сжатие по оси OX:

При сравнении графиков двух функций

\[y_1 = f(x), y_2 = f(px), p > 1\]

график второй функции сжимается в \(p\) раз по оси OX по сравнению с графиком первой функции. В частности для тригонометрических функций период второй функции уменьшается в \(p\) раз: \(T_2 = \frac{T_1}{p}\).

При сравнении графиков двух функций

\[y_1 = f(x), y_2 = f(\frac{x}{p}), p > 1\]

график второй функции растягивается в \(p\) раз по оси OX по сравнению с графиком первой функции. В частности для тригонометрических функций период второй функции увеличивается в \(p\) раз: \(T_2 = pT_1\).

Построим в одной системе координат три графика на интервале \(x \in [0; 2\pi]\):

\[f(x) = \sin(x), \space\space g(x) = \sin(2x), \space\space h(x) = \sin(\frac{x}{2})\]

Наибольшую гибкость и легкость в настройки нескольких графиков позволяет достичь последовательное декларирования графиков одним за другим используя plot(); заранее выставив опцию hold on;:

  • Период колебания функции \(g(x) = \sin 2x\) в два раза меньше: \(T_g = \frac{2 \pi}{2} = \pi\)
  • Период колебания функции \(h(x) = \sin \frac{x}{2}\) в два раза больше: \(T_h = 2 \cdot 2 \pi = 4\pi\)

Растяжение и сжатие по оси OY

При сравнении графиков двух функций:

\[y_1 = f(x), y_2 = Af(x), A > 1\]

график второй функции растягивается в A раз по оси OY по сравнению с графиком первой функции.

При сравнении графиков двух функций:

\[y_1 = f(x), y_2 = \frac{1}{A} f(x), A > 1\]

график второй функции сжимается в A раз по оси OY по сравнению с графиком первой функции.

В контексте синусоиды и косинусоиды говорят, что:

  • умножение на параметр \(A > 1\) увеличивает амплитуду колебаний в \(A\) раз;
  • деление на параметр \(A > 1\) уменьшает амплитуду колебаний в \(A\) раз.

Построим в одной системе координат три графика на интервале \(x \in [0; 2\pi]\):

\[f(x) = \sin(x), \space\space g(x) = 2\sin(x), \space\space h(x) = \frac{1}{2} \sin(x)\]

Параллельный перенос по оси OX

При сравнении графиков двух функций:

\[y_1 = f(x), y_2 = f(x + a), a > 0\]

график второй функции смещается влево на \(a\) по оси OX по сравнению с графиком первой функции.

При сравнении графиков двух функций:

\[y_1 = f(x), y_2 = f(x-a), a > 0\]

график второй функции смещается вправо на \(a\) по оси OX по сравнению с графиком перовй функции.

При работе с тригонометрическими функциями параметр \(x\) называют начальной фазой колебания. При сравнении двух тригонометрических функций \(y_1 = f(x)\) и \(y_2 = f(x \pm a)\) говорят, что у второй функции сдвиг по фазе равен \(\pm a\).

Общее уравнение синусоиды

\[y(x) = A \sin(cx + d) + B\]

где:

  • \(A\) - амплитуда, характеризует растяжение графика по оси OY
  • \(B\) - вертикальный сдвиг, характеризует сдвиг графика по оси OY (вверх/вниз)
  • \(c\) - циклическая частота, характеризует период колебаний и растяжение графика по оси OX
  • \(d\) - начальная фаза, характеризует сдвиг графика по оси OX (влево/вправо)

График \(y(x) = A cos(cx + d) + B\) также называется синусоидой. Термин «косинусоида» употребляется относительно редко. Поскольку график косинуса получается из графика синуса сдвигом по фазе на π/2 влево, вводить термин «косинусоида» излишне.

Построим график \(g(x) = 3 \sin (2x + \frac{\pi}{2}) - 1\)

По сравнению с \(f(x) = \sin(x)\):

  • \(A = 3\) - график растянут по оси OY в 3 раза
  • \(c = 2\) - период меньше в 2 раза \(T = \pi\), график сжат в 2 раза по оси OX
  • \(d = \frac{\pi}{2}\) - начальная фаза положительная, график сдвинут на \(\frac{\pi}{2}\) влево
  • \(B = -1\) - график сдвинут по оси OY на 1 вниз

  1. Более подробное объяснение ax = gca; ax.XAxis.FontSize = 14;

    • gca() или gca - это вызов функции которая возвращает текущие оси графика, при отсутсвии она их создаст.
    • ax = gca; - присваивается ссылка на текущие оси переменной под названием ax. Это позволяет использовать ax в последующих операциях для обращения к этим осям. Matlab\Octave не поддерживает цепочную точечную нотацию (англ: chained dot notation) на результате функции:
      • Неправильно: gca().XAxis.FontSize = 14; вернет ошибку error: invalid empty index list, Matlab воспринимает это как попытку доступа к некоторым элементам результата функции gca, ведь функцию gca можно вызвать и без скобок, если она не принимает аргумент: gca() \(\equiv\) gca.
      • Неправильно: gca.XAxis.FontSize = 14; - здесь Matlab не выполняет вызов функции, а воспринимает нотацию как создание структуры. Matlab создаст структуру с именем gca с полем XAxis, которое в свою очередь является структурой с полем FontSize, установленным в 14. Последнее поведение особенно коварно так как в памяти запишется новое значение gca, и предыдующие настройки графики будут уже обращаться к новой переменной gca при перезапуске скрипта, в таком случае необходимо удалить переменную из памяти окружения: >> clear gca;.
    • ax.XAxis: Этот код обращается к свойству XAxis ссылки на оси (ax). Свойство XAxis возвращает объект, представляющий ось X текущих осей, у которого есть свой набор свойств, которые можно изменить или запросить.
    • .FontSize: Это свойство объекта XAxis. Оно определяет размер шрифта меток на оси X.
    • ax.XAxis.FontSize = 14;: С помощью этой команды устанавливается размер шрифта меток на оси X текущих осей равным 14.
